## [0.1.0] - 2026-05-03

### Added

- `ab0b894` — Establish the project: ship the standard repo files (`.gitignore`, `.editorconfig`, `.markdownlint.jsonc`, `LICENSE`, `CHANGELOG.md`, `CLAUDE.md`, `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, `SECURITY.md`, `SUPPORT.md`, README skeleton) so contributors have a baseline to work from.
- `2e41b52` — Wire up the gates orchestrator so every PR can prove its quality bar in one command: `tests/gates/run-all.sh` discovers gate scripts under `tests/gates/`, runs them with a strict 0/1/2 exit contract, and writes per-gate NDJSON results under `tests/gates/.tmp/` for CI artefact upload. First three gates land alongside (`gate-01-doctor` skips until skeleton lands, `gate-02-no-absolute-paths`, `gate-03-shellcheck`).
- `53c22eb` — Make the package publishable and runnable end-to-end: `package.json` declares `@agentline/cli` with a single `agentline` bin, exact-version runtime deps, a strict `files` allowlist, and provenance-on publishing. `src/cli.ts` dispatches subcommands and falls through to a render path that reads stdin and prints a one-line ASCII fallback so the host UI is never blank. `src/stdin/` parses the Claude Code statusline payload safely (256 KB cap, truncation flag, untouched raw object, structured `StdinParseError` on bad input). `npm run {build,test,lint,typecheck,format}` are all wired; first vitest suite covers the parser.
- `354d925` — Give agentline a real configuration story: a layered loader merges built-in defaults, the user file, the per-project file, `AGENTLINE_*` env vars, and CLI flags (last layer wins, arrays replace wholesale). The merged tree is validated against a canonical JSON Schema so typos like `globel.padding` fail fast with every offending key listed. Persisted writes go through write-temp + fsync + rename so editor watchers always see one consistent state. The schema is embedded into the bin at build time so the render path needs zero filesystem reads. New `agentline schema [--write <dir>]` subcommand prints the schema or drops it into the user's editor config dir. Adds 33 unit tests (38 total passing).
- `8664c80` — Give widgets a colour vocabulary they can rely on: a theme is loaded from a small JSON file, validated against an embedded JSON Schema (Ajv 2020-12), and exposes thirteen named roles (`accent`, `info`, `success`, `warning`, `danger`, `muted`, the two `git-*`, the three `tokens-*`, and the two `bg-*`). Anything a theme leaves out falls back to a compiled default palette so the renderer can't end up colourless. Four presets ship — `vscode-dark`, `vscode-light`, `claude-code-dark`, `claude-code-light` — covering the two visual lineages most users come from. Colour values accept named ANSI, 256-colour indices (`colour:NNN`), and truecolor hex (`#RRGGBB`); validation runs after the schema check so a stray value can't reach the renderer. Loader is read-only and synchronous, with no network I/O. 39 new tests.
- `d3053f0` — Unblock the install/init/doctor/uninstall gate suite ahead of the real bodies: adds skeletons for `scripts/{install,init,doctor,uninstall}.sh` plus the shared `scripts/lib/common.sh` (logging, OS detect, Node ≥20 check, EXIT cleanup trap, guarded `al_safe_rm`) that parse documented flags and exit 0. Lands `gate-04-init-idempotency` (two consecutive runs against an isolated `$CLAUDE_PROJECT_DIR` sandbox must produce byte-identical snapshots) and flips `gate-01-doctor` from skip to pass.
- `ab6dad6` — Run the gate suite on every supported host on every PR and give the repo a consistent triage shape: adds `.github/workflows/gates.yml` and `install-matrix.yml` (macos-13/14, ubuntu-22.04/24.04, windows-2022 × Node 20/22; SHA-pinned actions; `permissions: read-all`; push-only concurrency cancellation; failure-artefact upload). Adds `.github/pull_request_template.md` mirroring PR-CONVENTIONS.md and three `.github/ISSUE_TEMPLATE/` forms (bug, feature, gate-failure). Lands `gate-05-markdown` (markdownlint-cli2 + prettier --check; tools independently skippable) and `gate-06-trademark` (allowlist-driven misuse check). Tunes `.markdownlint.jsonc` to disable MD012/MD040/MD060.
- `4cc7a76` — Lay down the render pipeline foundation every widget PR will build on. The renderer detects what the host terminal can actually display — `COLUMNS` env, then the Node tty ioctl, then a clean 80-column fallback — and applies the configured width mode (`full` / `full-minus-40` / `full-until-compact`) so compact-line behaviour is settled before any widget runs. Colour-depth detection follows the §8.3 contract (`COLORTERM=truecolor` → 24-bit; `*-256color` and `xterm-kitty` → 256; common `xterm` / `screen` / `tmux` / `vt100` → 16; `dumb` or unset → none). The ANSI encoder downgrades cleanly through cube quantisation for 256-colour and a nearest-named-RGB lookup for 16-colour terminals, and merges adjacent identically-styled segments into a single SGR run so the encoded line stays compact. Accessibility flags (`--no-color`, `--no-unicode`, `--ascii`) parse out of argv, the `NO_COLOR` env is honoured per the no-color.org convention, and a documented glyph table rewrites `·`, `✓`, `⚡`, the Powerline chevrons, and friends into ASCII when the user opts out of unicode. `writeOnce` finishes the frame in a single `stream.write(...)` so the host terminal never sees a torn line. 51 new tests.
- `eb3914e` — Give users a self-service way to verify and repair their Claude Code wiring: `agentline doctor` runs ten checks covering the settings file, the statusLine entry, the merged config, theme installation, Nerd Font availability under Powerline, git on PATH, pricing-table freshness, `CLAUDE_CONFIG_DIR` writability, custom-command resolution, and an embedded render dry-run. `--fix` repairs the four checks that are safe to auto-mutate (settings scaffold, statusLine wiring, config defaults, theme copy) and refuses to overwrite a foreign statusLine without `--force`. `--json` emits machine-readable output for CI; `--strict` flips the exit code so unresolved warnings/failures become non-zero. `scripts/doctor.sh` is the read-only shell wrapper that resolves the bin from local build, global install, or `npx` fallback. 13 new unit tests (50 total passing); all three repo gates green.
- `ace097c` — Make CI faithful to a real install: the gates workflow now runs `npm ci` (or `npm install`) and `npm run build` before invoking `tests/gates/run-all.sh`, so gate 01's doctor wrapper resolves the freshly built `dist/cli.mjs` instead of falling through to a non-existent published package and failing on every host.
- `3ff7c82` — Give first-run users a sensible statusline out of the box: ships `templates/default.config.json` (the nine-widget line — model, git branch / changes, context-percentage, total tokens, cost, session usage, flex separator, clock — with token / cost / session widgets bucketed by 5-hour block and the bundled `claude-code-dark` theme) and `templates/minimal.config.json` (model, git branch, flex separator, clock) for the `init --minimal` path. Both validate against the embedded JSON Schema; six new round-trip tests guarantee the templates and schema can never silently drift apart.
- `0f3d9f0` — Give every built-in widget a stable surface to register against. A widget is a small, frozen `WidgetDef` with a synchronous `render(ctx, settings)` returning a `Cell` (text + optional fg/bg + bold/italic + a `merged` mode that controls left-edge spacing + a `hidden` flag). The context bundles stdin, the merged config, the theme, a `Clock`, and the env snapshot so widgets never reach into ambient state — `frozenClock(at)` lets goldens reproduce timestamps to the millisecond. The registry is narrow on purpose (register / get / require / has / list / size) with structured errors for duplicate types and unknown lookups; built-in widgets in PRs 10–14 will register against `defaultRegistry()`. `renderWidget` is the bridge from `WidgetConfig` back to a frozen cell: `hidden: true` short-circuits, `fg` / `bg` / `bold` / `italic` overrides win when set, `merged` defaults to `off`, and `rawValue` is threaded through `WidgetSettings` so widgets that emit a label can suppress it without callers mutating options. The loose config `Colour` string is bridged to the strict encoder `Colour` at this boundary so the encoder keeps its narrow types. 33 new tests.
- `d5e1676` — Eliminate the per-PR `CHANGELOG.md` conflict that was costing every open branch a manual rebase, and write down the maintenance contract that goes with it: introduces a `changelog/<NN>-<slug>.md` fragment per PR, a `scripts/changelog-aggregate.sh` aggregator (dry-run by default; `--apply` rewrites `[Unreleased]` and removes the consumed fragments) with optional `<!-- sha: ... -->` hints for backfills, extends `docs/plan/PR-CONVENTIONS.md` with both the fragment workflow and a new "Author maintenance" section requiring the author to keep CI green and rebase out merge conflicts before review, and migrates every existing `[Unreleased]` bullet (bootstrap, gates orchestrator, TS scaffold, config loader, lifecycle skeletons, CI workflows, theme engine) into individual fragment files so the aggregator becomes the single source of truth.
- `e8acd67` — Make agentline a one-command install: `scripts/install.sh` now installs (or `npm link`s with `--from-source`) the package, seeds the user config from the bundled default, copies the four shipped themes into the user's themes dir, and writes a working `statusLine` entry into Claude Code's settings file. It's idempotent (re-runs are a byte-for-byte no-op), `--dry-run` describes every action without touching disk, and it refuses to overwrite a non-agentline `statusLine` unless `--force` is set. `scripts/uninstall.sh` mirrors the cleanup: only removes themes / config that still byte-match the shipped originals (preserving user edits), strips the `statusLine` entry only when it still references agentline, and supports `--purge` and `--dry-run`. Thirteen new integration tests cover the round-trip, idempotency, dry-run parity, foreign-statusLine refusal, and `--purge` behaviour.
- `b46bb96` — Land the eleven session widgets — `model`, `version`, `output-style`, `session-id`, `session-name`, `account-email`, `login-method`, `org`, `thinking-effort`, `vim-mode`, and `skills` — plus the auth-file fallback they share. The render-tick resolver reads `stdin.user.*` first and only drops to the auth file (CLAUDE_CONFIG_DIR-aware, with a per-platform default) when stdin omits an identity field; the read is sync, read-only, and never throws so a missing or unreadable file simply hides the dependent widget. Widgets see a flat `ctx.session` record and never touch the filesystem themselves, keeping the render hot path I/O-free. `model` maps known Anthropic ids to friendly names (Opus 4.7, Sonnet 4.6, Haiku 4.5, …) and falls back to the raw id otherwise. `account-email` supports `none` / `domain` / `localpart` masks for screenshot-friendly output. `thinking-effort` colour-grades against the theme palette so `low` / `medium` / `high` / `xhigh` map cleanly to success / info / warning / danger. `vim-mode` exposes long / short / bracket formats and `skills` exposes count / list / last variants — both keyboard-cyclable in the TUI. Every widget honours `rawValue` (label suppression per §5.3) and hides cleanly when its source is absent. 56 new tests.
- `a431cc9` — Prove every release works on every supported host before it ships: the install-matrix workflow now packs the repo with `npm pack` on macos-13/14, ubuntu-22.04/24.04, and windows-2022 × Node 20/22, installs the resulting tarball into an isolated `--prefix`, pipes a canonical Claude Code statusline payload through the freshly installed bin, and asserts the exact stdout — so a publish-time regression in packaging, bin resolution, or the render path is caught on every PR. The same recipe lands locally as `tests/gates/gate-15-platform-matrix.sh` so contributors can validate before pushing. The workflow then runs `install.sh && uninstall.sh` against a temp `HOME` / `CLAUDE_CONFIG_DIR` to keep gates 07/09/10 honest, and uploads `.ci-pack/*.tgz` plus `tests/gates/.tmp/` as failure artefacts for triage.
- `76c83f0` — Land the eight token / cost / speed widgets and the four context widgets, plus the data layer they share. The new `src/tokens/` module embeds a versioned USD-per-Mtoken pricing table (whose version is exposed to `agentline doctor`), a per-model context-window lookup (200k for the standard families, 1M for the `[1m]` variant), and a JSONL transcript reader with a process-wide cache keyed by `(path, mtime, size)` and the §O3 eviction policy (5 h idle / 32 MB total). The render-tick resolver `loadTokensSnapshot()` reads the transcript once and freezes a `TokensSnapshot` widgets index into; widgets stay pure and never touch the filesystem during `render()`. `tokens-input`, `tokens-output`, `tokens-cached`, and `tokens-total` each declare a reset axis (`session` / `block` / `day` / `week` / `model` / `effort`) that filters events before summing — mixed-axis aggregation is forbidden per §8.4, and a typo on `options.reset` falls back to `session` so the renderer can never crash on a stray value. `cost` prices each model independently inside the window so a block that mixes Sonnet and Haiku is priced correctly. `input-speed`, `output-speed`, and `total-speed` report rolling tokens/second over a 60 s window (configurable via `options.windowSec`, capped at one hour). `context-length` reports the session's used+cached count, `context-percentage` and `context-percentage-usable` divide by the model's window (or 80% of it) and colour-grade against the `tokens-low / tokens-mid / tokens-high` palette roles at the 60 / 80% boundaries, and `context-bar` renders a fixed-width bar (default 12, configurable filled/empty glyphs) coloured by the same roles. 68 new tests; 302 total passing.
- `4c06e15` — Verification: `gate-11-schema-roundtrip` validates the embedded JSON Schema against the shipped config templates so they cannot drift apart silently.
- `befa75f` — Verification: `gate-14-no-network-render` runs the render path under a network-blocking sandbox (`sandbox-exec` on macOS, `unshare -nr` on Linux) so any outbound call from the hot path is caught immediately.
- `32802c7` — Document the consumer surface: `docs/install.md`, `docs/config.md`, `docs/widgets.md`, `docs/themes.md`, `docs/keymap.md`, and `docs/doctor.md` cover the install + manual recipes, the layered config and JSON Schema, the built-in widget catalogue with reset axes, the four shipped themes plus Powerline, the TUI editor keymap and overrides, and the full D01–D10 doctor checks with exit codes.
- `228ad08` — Verification: `gate-16-accessibility-fallbacks` exercises `--no-color`, `--no-unicode`, `--ascii`, and `NO_COLOR=1`, asserting each flag drops the bytes it advertises so the renderer can't quietly leak ANSI or non-ASCII into accessibility-constrained terminals.
- `044b5b0` — Replace the one-line `README.md` with a real landing page: CI / npm / Node / license badges, a one-command install (`npm install -g @agentline/cli && agentline doctor --fix`), the `scripts/install.sh` flow, configure / themes / verify recipes, links to every page under `docs/`, and an explicit v0.1.0 non-goals section so the CLI-only scope is unmissable.
- `257e700` — Verification: `gate-18-changelog-fragment-present` blocks PRs that ship non-doc changes without dropping a `changelog/<NN>-<slug>.md` fragment, so the fragment-per-PR convention is enforced by CI rather than by reviewer memory.
- `917ee12` — Surface block / weekly quota usage and reset countdowns in the statusline: ship the eight rate-limit widgets — `session-usage`, `weekly-usage`, `block-timer`, `block-reset-timer`, `weekly-reset-timer`, `model-usage`, `effort-usage`, `compaction-counter` — reading the existing transcript snapshot and reusing the token / week / block axes.
- `48f6e5c` — Reviewer affordance: a `changelog-preview` workflow runs `scripts/changelog-aggregate.sh --dry-run` on every PR and posts the next-release preview as a sticky comment that updates in place, so reviewers can read the aggregated bullets without opening every fragment file.
- `6a3c989` — Surface git working-tree state in the statusline: ship the 16 git widgets — branch, changes / insertions / deletions, status / staged / unstaged / untracked, ahead-behind, conflicts, sha, worktree, origin owner / repo, upstream, is-fork — backed by a single `git -C <cwd>` snapshot loader with CRLF normalisation that hides every widget outside a git repo.
- `97d575b` — Round out the built-in widget set: ship the time family (`clock` with configurable format, `uptime-session`, `uptime-block`) and the custom family (`separator`, `flex-separator` with a renderer-recognised flex marker, sandboxed `command` with timeout / cache / byteLimit / onError marker).
- `54775ec` — Render the statusline with Powerline triangles or the ASCII fallback: ship the Powerline transform (chevron insertion with adjoining-colour rule, autoAlign, continueColors, glyph overrides, Nerd-Font/ASCII selection) plus a plain-mode line composer that expands flex-separator slots to fill remaining width and treats them as a no-op when Powerline mode is enabled.
