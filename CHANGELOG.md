## [Unreleased]

### Changed

- `a99a7a6` — Compress the CLI surface and add an onboarding skill: top-level is now four verbs — `install`, `uninstall`, `doctor`, `config` — plus the default render path; `init`, `theme`, `keys`, and `schema` move under `agentline config <sub>` so the top-level help stays small; bare `agentline config` still opens the TUI editor; `agentline preview` is removed entirely (install is reversible, so the trial path is "install, ask the in-session agent, uninstall to revert"); a new `agents/agentline-onboarding.md` skill ships alongside the four existing skills, so `agentline install` now seeds five `agentline*.md` skills into `$HOME/.claude/agents/` for in-session configuration with Claude Code without leaving the session. Breaking renames: `agentline init` → `agentline config init`, `agentline themes` → `agentline config theme`, `agentline keys` → `agentline config keys`, `agentline schema` → `agentline config schema`.
- `PENDING` — Rationalise init presets and turn theme selection into a single command. `agentline config init` now ships exactly three presets: `minimal` (model + `context-length` + `block-reset-timer` — what model, how much context is left, when the 5h block resets), `default` (the previous balanced bar), and `maximal` (a curated, single-line "everything useful": thinking-effort, git branch + changes, context, tokens, cost, session + weekly usage, block + weekly reset timers, clock); `focus` and `power` templates are dropped and the undocumented `--minimal` alias is removed. `agentline config theme` gains `--set <name> [--scope user|project]`, which validates the theme against the search path and immutably patches `theme: "<name>"` into the chosen config via the same atomic-write path as `init` (creating a `{version:1, theme:<name>}` file if none exists). Help text for both commands is rewritten to spell out the browse → pick → verify flow so users no longer have to hand-edit JSON to switch themes.
