- Add an npm-registry update-check cache surfaced through `agentline doctor`. `install` and `edit` fire a fire-and-forget probe of `https://registry.npmjs.org/@agentline/cli/latest` (3s timeout, never throws), persist the result atomically under `${agentlineDir}/state/version-check.json`, and skip the network call for 24h if the cache is fresh. `agentline doctor` gains a new `D09` check that reads the cache and reports `update available: <current> → <latest>`, `up to date`, `last probe failed`, or `no cached check yet`. The render hot path stays untouched — no network, no cache read — so gate-14 keeps its invariant. A `semver`-aware `isNewer` comparator decides when to surface the upgrade hint, treating prereleases as lower priority than the matching stable triplet.
