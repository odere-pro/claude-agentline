Give widgets a colour vocabulary they can rely on: a theme is loaded from a small JSON file, validated against an embedded JSON Schema (Ajv 2020-12), and exposes thirteen named roles (`accent`, `info`, `success`, `warning`, `danger`, `muted`, the two `git-*`, the three `tokens-*`, and the two `bg-*`). Anything a theme leaves out falls back to a compiled default palette so the renderer can't end up colourless. Four presets ship — `vscode-dark`, `vscode-light`, `claude-code-dark`, `claude-code-light` — covering the two visual lineages most users come from. Colour values accept named ANSI, 256-colour indices (`colour:NNN`), and truecolor hex (`#RRGGBB`); validation runs after the schema check so a stray value can't reach the renderer. Loader is read-only and synchronous, with no network I/O. 39 new tests.
