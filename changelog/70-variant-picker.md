- `agentline config` now uses a three-step drill-down picker (group → widget → variant) and a new `u` (update) verb that switches the selected widget's variant in place — same widget, different rendering. Variants are a new optional `variants` field on each `WIDGET_CATALOG` entry: each variant declares `{ id, label, options }` where `options` is the patch merged into `WidgetConfig.options` when the user picks it. Populated for the widgets that already branch on an option — `skills` (count / list / last), `session-usage` and `weekly-usage` (percent / bar / short-bar), `vim-mode` (long / short / bracket), `account-email` (full / domain / localpart), `block-timer` / `block-reset-timer` / `weekly-reset-timer` / `uptime-session` / `uptime-block` (short / long / clock), and `clock` (five common strftime presets). Widgets without distinct rendering modes skip step 3 and commit straight from step 2 with default options. The picker UX: step 1 shows categories with glyphs + widget counts, step 2 lists widgets in the chosen group with a live filter and per-row mini-preview, step 3 shows each variant with a live mini-preview (the same widget rendered against the demo session, once per variant) and a synthetic first row — "Keep current options" for `update`, "Default options" for `add`/`replace`. <kbd>Esc</kbd> steps back one level (cancels at step 1); for an `update` target, stepping back from step 3 lands straight on edit (steps 1 and 2 were skipped). New plumbing: `src/widgets/catalog.ts` adds `WidgetVariant`, the optional `variants` field, and `widgetVariants(type)` / `activeVariantId(type, options)` helpers; `src/tui/state.ts` rewires modes (`picker-group` / `picker-widget` / `picker-variant`), the structured `PickerTarget { kind, line, index }`, a `PickerDraft`, and the new actions (`open-picker {intent}`, `open-update`, `pick-category`, `pick-widget`, `pick-variant`, `picker-back`, `close-picker`) — `pick-widget` commits immediately for no-variant widgets, the variant `options` patch is sanitised on commit to drop prototype-polluting keys; `src/tui/picker.ts` is rewritten as the three step components (`PickerGroup` / `PickerWidget` / `PickerVariant`) plus pure helpers (`categoriesWithWidgets`, `widgetsInCategory`, `variantRows`, `selectedAt`); `src/tui/main.ts` routes the overlay per mode, resets per-step query/highlight on every transition, and surfaces a status message when `u` is pressed on a widget without variants. `src/keys/bindings.ts` adds `update` and `picker-back` action ids (and drops the redundant `picker-cancel` — `picker-back` covers both stepping back and cancelling from step 1); `bindings.test.ts`, `tests/gates/gate-17-keymap-coverage.sh`, `docs/keymap.md`, and `docs/plan/SPEC-v0.1.0.md` §5.5 are updated in lockstep.
