- Lift the four mode-specific keypress handlers out of `src/tui/use-editor-input.ts` into a new pure module `src/tui/editor-input-handlers.ts`. Each `handle*Key(input, key, deps)` is now a plain function (no `useCallback` closures over half the editor state); every side effect — dispatch, step-state setters, exit, save, status message — flows through a callback in the typed `deps` bag. The hook keeps `stepQuery` / `stepHighlight` ownership and the `useEffect` that resets them on mode change; everything else routes via a tight `switch (state.mode)`.
- `src/tui/use-editor-input.ts` drops from 322 LOC to 118 LOC. The four handler bodies that were 62 / 47 / 34 / 66 LOC inside `useCallback` blocks (with eight-symbol dependency arrays each) are now flat top-level functions; `useInput` re-renders no longer churn the callback identities. Closes smell-report 2026-05-15 `H-code-1`.
