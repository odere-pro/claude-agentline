- Add four install/uninstall shell gates (gate-07 … gate-10), wired into `tests/gates/run-all.sh`, promoting the install round-trip invariants from the slow integration test onto the host matrix. Each runs in a hermetic sandbox (temp `HOME` + `CLAUDE_CONFIG_DIR` + an `agentline` shim on `PATH` so the global `npm install` step is skipped — offline, never touches the real `~/.claude`); shared harness in `tests/gates/lib/install-sandbox.sh`:
  - **gate-07 — round-trip:** install wires `statusLine` to agentline; uninstall restores a pre-existing foreign `statusLine` byte-for-byte and leaves no agentline config/theme footprint.
  - **gate-08 — content preservation:** install/uninstall touch only `statusLine`; every other settings key (`theme`, `hooks`, `env`, `permissions`) is byte-identical across the round-trip.
  - **gate-09 — idempotency:** a second `install` is a byte-clean no-op (tree **and** file-content hashes identical).
  - **gate-10 — dry-run parity:** `install --dry-run` changes nothing on disk while still reporting the actions it would take.
- **fix(install): make the install manifest idempotent.** gate-09 surfaced a real non-idempotency — `scripts/install.sh write_manifest` stamped a fresh `installedAt` timestamp on every run, so a second install rewrote `state/manifest.json`. It now preserves the original `installedAt` on re-install and skips the write when the manifest body is unchanged, so a re-install is a true no-op. (The prior integration test only snapshotted the tree shape, not file content, so it never caught this.)
