- Fix the reset countdown/clock so it matches the host's `/usage` screen: `current-session-reset-timer`, `current-session-reset-at`, `week-limit-timer`, and `weekly-reset-at` now read the host's `rate_limits.{five_hour,seven_day}.resets_at` off stdin instead of guessing the reset from the local 5-hour block anchor or a default Monday-00:00 weekly anchor (which produced wrong values like `reset in 4h 55m` / `reset in 5d 23h 23m`).
