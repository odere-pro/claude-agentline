- Editor picker step 1 now hosts a search field on top of the group list: typing flips the view to a flat substring-filtered list across every category (with per-row category accents and badges), Enter on a flat result commits the widget directly (skipping the in-category step), and Backspace clears back to the group view; the existing category drill-down stays available when the search is empty. Already-added widgets continue to be hidden from both views via the `usedTypes` exclude set, with a carve-out for `custom`-category widgets like `separator` so they can repeat. Reducer-side: `pickWidget` now accepts dispatch from `picker-group` (flat-search path) in addition to `picker-widget`, and `picker-back` from `picker-variant` returns to whichever step the user came from (`picker-group` if no category was chosen, `picker-widget` otherwise).
