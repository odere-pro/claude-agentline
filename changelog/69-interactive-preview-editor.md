- The `agentline config` TUI editor is now organised around the live preview as the editing surface (§1.1 F10). All three line slots are rendered at all times; the selected widget is highlighted in place (Ink `inverse` plus `‹…›` brackets so the cue survives `NO_COLOR`); each row ends in a navigable **+ add widget** cell that the cursor walks onto and confirms to insert. The previously-separate "layout list" view is gone — the cursor moves through the rendered statusline itself. **Fixes the up/down arrows**, which were no-ops on single-row configs because the editor didn't pad to the full grid. New plumbing: `src/tui/state.ts` reworks the cursor to a `0..widgets.length` model (the `len` index is the add-cell; `currentWidget` returns `undefined` there; the `-1` sentinel is gone), pads `state.lines` to exactly `MAX_LINES` in `initialState`, and clamps movement onto the add-cell column; `src/tui/persist.ts` trims trailing empty rows on write so the padding stays invisible to the on-disk config. `src/tui/preview-model.ts` (new, pure, no Ink) builds a structured row model with per-cell provenance — widget / join / add slots — that `src/tui/preview.ts` projects into Ink with the cursor highlighted in place; hidden widgets render as a dim `[hidden:type]` chip so they remain navigable. `src/tui/glyphs.ts` (new) ships an editor glyph set (`‹›`, `▸`, `│`, `＋ add widget`, per-category icons) with an ASCII fallback when the locale isn't UTF-8. `src/tui/main.ts` rewires `useInput`: <kbd>↵</kbd> on the +add cell opens the picker, <kbd>↵</kbd> on a widget opens the options sheet; <kbd>d</kbd> deletes (<kbd>x</kbd> kept as a backstop); Ctrl+S also saves alongside <kbd>S</kbd>. `src/keys/bindings.ts` adds the `edit-widget` action and switches the displayed delete key from `x` to `d`; `docs/keymap.md` is rewritten for the new model. The render path is unaffected — none of `src/render/*` changes, the golden harness stays green, and tsup's split keeps the new modules out of `dist/cli.mjs` (`grep -c "isAddCell\|buildPreview\|pickGlyphs" dist/cli.mjs` → 0). The Phase 1 theme threading is incorporated into the rewrite so the eventual rebase resolves cleanly.
