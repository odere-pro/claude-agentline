Land the eleven session widgets — `model`, `version`, `output-style`, `session-id`, `session-name`, `account-email`, `login-method`, `org`, `thinking-effort`, `vim-mode`, and `skills` — plus the auth-file fallback they share. The render-tick resolver reads `stdin.user.*` first and only drops to the auth file (CLAUDE_CONFIG_DIR-aware, with a per-platform default) when stdin omits an identity field; the read is sync, read-only, and never throws so a missing or unreadable file simply hides the dependent widget. Widgets see a flat `ctx.session` record and never touch the filesystem themselves, keeping the render hot path I/O-free. `model` maps known Anthropic ids to friendly names (Opus 4.7, Sonnet 4.6, Haiku 4.5, …) and falls back to the raw id otherwise. `account-email` supports `none` / `domain` / `localpart` masks for screenshot-friendly output. `thinking-effort` colour-grades against the theme palette so `low` / `medium` / `high` / `xhigh` map cleanly to success / info / warning / danger. `vim-mode` exposes long / short / bracket formats and `skills` exposes count / list / last variants — both keyboard-cyclable in the TUI. Every widget honours `rawValue` (label suppression per §5.3) and hides cleanly when its source is absent. 56 new tests.
