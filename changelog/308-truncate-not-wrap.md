- fix: a configured statusline line now always occupies exactly one physical row. The plain composer used to wrap overflow onto extra rows, so a 3-line config emitted 9/6/6/4/3 rows at COLUMNS 60/80/100/120/140. Claude Code paints one terminal row per newline-separated segment of the statusline command's stdout, so that varying row count broke its erase-and-redraw accounting and leaked stale statusline copies into the scrollback whenever ordinary content drift (a cost value gaining a digit, a timer counting down, a PR number appearing) crossed a wrap boundary between refreshes. Overflow is now elided at a cell boundary and marked with an ellipsis (`...` when the host cannot render unicode), never mid-cell and never silently.
