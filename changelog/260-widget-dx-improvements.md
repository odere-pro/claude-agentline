- Make "configured but invisible" widgets discoverable: add doctor check **D11 (widget config sanity)**. It warns when the config names a widget `type` that is not in the registry (e.g. a type removed in an upgrade) or a `git-pr` widget without `options.allowNetwork`, both of which render nothing silently. Clean configs report `ok`. The valid-type list is derived from the catalogue, never hardcoded.
- Harden `account-email` against showing a mismatched identity: when the session's stdin declares a user identity, the on-disk `auth.json` / `~/.claude.json` email is no longer used as a cross-source fallback (hide over mislead). The common case — no stdin user block — still falls back to the auth file unchanged.
