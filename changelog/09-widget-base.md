Give every built-in widget a stable surface to register against. A widget is a small, frozen `WidgetDef` with a synchronous `render(ctx, settings)` returning a `Cell` (text + optional fg/bg + bold/italic + a `merged` mode that controls left-edge spacing + a `hidden` flag). The context bundles stdin, the merged config, the theme, a `Clock`, and the env snapshot so widgets never reach into ambient state — `frozenClock(at)` lets goldens reproduce timestamps to the millisecond. The registry is narrow on purpose (register / get / require / has / list / size) with structured errors for duplicate types and unknown lookups; built-in widgets in PRs 10–14 will register against `defaultRegistry()`. `renderWidget` is the bridge from `WidgetConfig` back to a frozen cell: `hidden: true` short-circuits, `fg` / `bg` / `bold` / `italic` overrides win when set, `merged` defaults to `off`, and `rawValue` is threaded through `WidgetSettings` so widgets that emit a label can suppress it without callers mutating options. The loose config `Colour` string is bridged to the strict encoder `Colour` at this boundary so the encoder keeps its narrow types. 33 new tests.
