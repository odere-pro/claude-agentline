- Tighten config/schema correctness so the shipped schema matches reality and the authoring surface rejects mistakes instead of silently coercing them:
  - **`$schema` / `$id` URLs now resolve.** The config and theme schema `$id`s and every template/theme `$schema` reference moved from `github.com/.../schemas/...` (a 404 for a raw fetch) to `https://raw.githubusercontent.com/odere-pro/claude-agentline/main/schemas/...`.
  - **`widget.type` is now a catalogue-derived `enum`**, generated at build time from `WIDGET_CATALOG` by `scripts/gen-schema-enum.mjs` (wired as a `prebuild` step). Adding or removing a widget regenerates the enum with no hand-edit; **gate-28** (plus a vitest mirror) keeps the checked-in schema in sync.
  - **`version` is pinned to `enum: [1]`** (was `minimum: 1`) — an unknown future schema version is refused, not half-migrated (D-007).
  - **`label` is documented as the universal widget option** on `widget.options` (the code reads `settings.options.label`); no fictional top-level `label` property was invented.
  - **Mutation-time option validation.** `config widget add --options …` / `set-option` now reject an unknown option key (`set-option notakey foo`) or an out-of-range value (`--options '{"reset":"bad"}'`) with a helpful, catalogue-pointing message and a non-zero exit, instead of writing a silently-ignored option. Validation is catalogue-derived (`src/widgets/families/option-spec/option-spec.ts`) and shared by the CLI and the TUI editor through the one mutation path.
