- Make config failures self-healing and visible instead of silent:
  - **`doctor --fix` no longer crashes** when `settings.json` is missing _and_ `config.json` is corrupt at the same time (the most likely fresh-install-on-a-dirty-machine state). `fixD02`/`fixD09` no longer let a `loadConfig` failure escape; they fall back to the default refresh interval and the one-pass fix sequence converges (`fixD03` reseeds the config). D09/D11 degrade gracefully to "config not loaded — see D03".
  - **The live render no longer silently swallows an invalid config.** When a config file exists but is malformed JSON or fails schema validation, the render still falls back to defaults (the statusline is never blank) but now emits a one-line `agentline: config invalid (using defaults) — run \`agentline doctor\``diagnostic to stderr (TTY only, suppressed under`AGENTLINE_QUIET`). A merely-absent config stays silent (first-run path unchanged).
  - **Doctor D03 reports the full validation error**, including the offending field path, instead of truncating at the first line.
