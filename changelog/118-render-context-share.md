- Extract the live-snapshot loader and the `WidgetContext` builder into a single `src/render/context.ts`: `loadLiveSnapshots(payload, opts?)` and `buildWidgetContext(input)` now back both the live render path (`src/render/pipeline.ts` + `src/render/fixture-command.ts`) and the editor preview (`src/tui/preview-fixture.ts`). The two sites had been hand-rolling the same `{ session, tokens, git }` triple from the same loaders independently — a stovepipe the smell-report flagged as the load-bearing arch-medium dup.
- Relocate `RenderInputs` from `src/render/pipeline.ts` to a dedicated `src/render/inputs.ts` module. The pipeline keeps a `export type { RenderInputs }` re-export so existing imports (`pipeline.test.ts`) stay valid, but the pipeline's direct import surface drops `TokensSnapshot`, `GitState`, `ResolvedSessionFields`, `StdinPayload`, and `WidgetContext` — those types now live only on the bag definition in `inputs.ts`. The pipeline's cross-module import count drops from 9 distinct directories to 6.
- Net result: `pipeline.ts` body uses `buildWidgetContext(...)` instead of the previous 9-line inline assembly; `fixture-command.ts#loadLiveSnapshots` is replaced by a 5-line wrapper that delegates to the shared helper; `preview-fixture.ts#computePreviewMode` is now 4 lines instead of 14. Future additions to the snapshot triple (e.g. a workspace snapshot) land in one place.
