- Add the first two mutating `agentline config widget` subcommands: `agentline config widget add <type> [--line N] [--at I] [--options JSON]` inserts a built-in widget (`--line` defaults to 0 and pads empty lines up to it; `--at` defaults to the end of the line; `--options` takes a JSON object — prototype-polluting keys are rejected); `agentline config widget remove [--line N] --at I` drops the widget at that position (the emptied line is kept). Both go through `saveAddWidget` / `saveRemoveWidget` (`loadConfig` → mutate → validate → atomic write to the user config) and print a single confirmation line to stdout; bad arguments and out-of-range / unknown-type errors print to stderr and exit 2. The `config widget` help now documents them; `move` / `replace` / `set-option` are still to come.
