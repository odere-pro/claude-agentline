- Add a widget catalogue: `src/widgets/catalog.ts` defines a `WidgetMeta` (`name`, `description`, `category`, optional `previewFixture`) for all 53 built-in widgets and exposes `WIDGET_CATALOG` / `widgetMeta(type)`; `WidgetRegistry` gains `listMeta()` returning the catalogued entries for the registered types, sorted by `type`. `categories` mirror the source-tree families (`session` / `tokens` / `context` / `rate-limits` / `git` / `time` / `custom`). The render-path `WidgetDef` contract is unchanged — metadata lives in one auditable place rather than threaded through every `defineWidget` call — and a new `catalog.test.ts` enforces the invariants (one entry per built-in, no orphans, non-empty descriptions ≤ 80 chars, known categories). Groundwork for the TUI widget picker and the planned `agentline config widget catalog` command.
