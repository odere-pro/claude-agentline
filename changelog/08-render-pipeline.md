Lay down the render pipeline foundation every widget PR will build on. The renderer detects what the host terminal can actually display — `COLUMNS` env, then the Node tty ioctl, then a clean 80-column fallback — and applies the configured width mode (`full` / `full-minus-40` / `full-until-compact`) so compact-line behaviour is settled before any widget runs. Colour-depth detection follows the §8.3 contract (`COLORTERM=truecolor` → 24-bit; `*-256color` and `xterm-kitty` → 256; common `xterm` / `screen` / `tmux` / `vt100` → 16; `dumb` or unset → none). The ANSI encoder downgrades cleanly through cube quantisation for 256-colour and a nearest-named-RGB lookup for 16-colour terminals, and merges adjacent identically-styled segments into a single SGR run so the encoded line stays compact. Accessibility flags (`--no-color`, `--no-unicode`, `--ascii`) parse out of argv, the `NO_COLOR` env is honoured per the no-color.org convention, and a documented glyph table rewrites `·`, `✓`, `⚡`, the Powerline chevrons, and friends into ASCII when the user opts out of unicode. `writeOnce` finishes the frame in a single `stream.write(...)` so the host terminal never sees a torn line. 51 new tests.
