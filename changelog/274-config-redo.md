- Make `agentline config undo` reversible and add `agentline config redo` (extends the G5 single-slot undo into a small 2-slot stack):
  - The config-backup seam (`src/data/config/backup/`) now keeps two slots beside the config — `config.json.bak` (the back/undo slot, unchanged) and `config.json.redo` (the forward/redo slot). `config undo` captures the pre-undo config into the forward slot before restoring the back slot, so `config redo` can roll it forward again; `redo` is the symmetric mirror (capturing the pre-redo config into the back slot), making `undo → redo → undo` a byte-identical toggle.
  - **A new edit after an undo invalidates the redo.** Every config-writing path (`config widget add/remove/move/replace/set-option` and a TUI editor save) drops the forward slot at the one shared write seam — you cannot redo into a branch you have diverged from (standard undo/redo semantics).
  - One step back, one forward — not a multi-level history (KISS). `config undo` / `config redo` each print a one-line confirmation that points at the inverse verb; "nothing to undo" / "nothing to redo" exit non-zero with a clear message rather than crashing; `--help` prints help and does not act.
  - Slots are written through the one atomic-write helper (temp+fsync+rename); slot deletes tolerate an absent file.
