- fix: `thinking-effort` no longer infers `ultracode` from a reported `xhigh`. The host keeps `ultracode` as a separate boolean it never puts in the statusline payload, so `/effort xhigh` and `/effort ultracode` are indistinguishable; and `effort.level` is a host-_resolved_ value (launch-effort pin, `CLAUDE_CODE_EFFORT_LEVEL`, `xhigh`→`high` capability downgrade), so an ultracode session on Opus 4.8 actually reports `high`. The default-on relabel shipped in 1.6.1 was therefore wrong in both directions — mislabelling plain `xhigh` sessions and missing real ultracode ones. `assumeUltracode` reverts to opt-in (the `literal` opt-out variant becomes an `ultracode` opt-in variant), the shipped templates drop the option, and `agentline start` shows a one-time correction notice.
