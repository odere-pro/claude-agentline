- Opt-in Nerd Font glyph layer. New top-level `glyphs: "off" | "nerd-font"` config field (default `"off"`) controls whether each widget's catalogue glyph is prepended to its rendered cell text. When `"nerd-font"`, `renderWidget` prepends `<glyph><U+2009 thin space>` to non-empty cells using the codepoint from the new `WIDGET_CATALOG[type].glyph`; flex separators, layout-only widgets, and widgets without a catalogued glyph are unaffected. Defaults stay `"off"` so the golden harness stays byte-stable. Plumbing: `src/config/types.ts` adds the `GlyphMode` type, `src/config/defaults.ts` seeds `"off"`, the JSON schema declares the enum (`additionalProperties:false` would have rejected the new key otherwise), `src/widgets/catalog.ts` adds `glyph?: string` to `WidgetMeta` plus a separate `WIDGET_GLYPHS` table merged into the canonical catalogue at module load (keeps the entry list scannable), `src/widgets/render-widget.ts` reads `ctx.config.glyphs` once per render and prepends conditionally — no per-widget changes required. The editor's live preview honours the mode via a new `glyphs?` field on `DemoOptions`/`previewWidget` and a one-line thread through `preview-model.ts#renderSlot`. 47 widgets ship with a glyph; layout-only `separator` / `flex-separator` deliberately do not. 13 new tests across `catalog.test.ts` (every glyph is a single grapheme; populated for the most-shipped widgets), `render-widget.test.ts` (off / on / unknown / flex / empty paths), `defaults.test.ts` (default is `"off"`), `validate.test.ts` (enum validation), and `demo-fixture.test.ts` (preview + override).
