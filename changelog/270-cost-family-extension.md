- Add three derived cost widgets to the tokens family, fed by the already-parsed stdin `cost` block (catalogue 34 → 37). Like `cost-usd`, they read host pre-computed scalars and carry no reset axis:
  - **`cost-burn-rate`** — session spend rate, `$/hr` = `totalUsd ÷ (totalDurationMs / 3.6e6)`, e.g. `$1.20/hr`. Hides when `totalUsd` or `totalDurationMs` is absent, or the duration is 0 (no divide-by-zero).
  - **`api-duration`** — time spent waiting on the API, from `apiDurationMs`. Absolute by default (`2.3s`, `1m 5s`); `options.percent` renders the share of wall-clock (`apiDurationMs ÷ totalDurationMs`) as a percent, hiding when the wall scalar is absent or 0.
  - **`cost-efficiency`** — API-active ratio `apiDurationMs ÷ totalDurationMs` as a percent (capped at 100%), e.g. `70%`. Hides when either field is absent or the duration is 0.
- These live alongside `cost-usd` in the `tokens` family — no new family (host cost scalars do not warrant a dedicated family or its shared-contract cost). The USD part of the burn rate reuses `formatUsd`; a new seconds-aware `formatCostDuration` joins `tokens/format/` for `api-duration` (the rate-limit timer formatters floor to whole minutes, so a 2.3s API call would have read as `0m`).
- All three are pure `(ctx, settings) → Cell`, hide when their source field is absent, and are opt-in (not in any default fixture) so the render goldens are unchanged. The `widget.type` schema enum regenerates from the catalogue via the prebuild gen script (gate-28); README / glossary / `docs/widgets.md` counts updated.
