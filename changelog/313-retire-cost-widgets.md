- feat!: retire the `cost-burn-rate` and `cost-efficiency` widgets. Both divided by `cost.total_duration_ms`, which is the host's idle-inclusive lifetime wall-clock — one real capture reads 118 hours. So `cost-burn-rate` reported a lifetime average diluted by overnight idle (two captures of one session 29 minutes apart show a real burn of `$4.24/hr` rendered as `$0.58/hr`), and spiked absurdly on short sessions instead; while `cost-efficiency` reported the fraction of wall-clock spent blocked in an API call, which ranged 1.2%–95% across captures purely as a function of how long the terminal sat open, and said nothing about cost despite its name. The honest half survives: `api-duration` with `options.percent` reports the same API-active share under a name that describes it. Thanks to the validation fix in #312, an existing config naming either type keeps rendering every other cell — only the retired ones are omitted — and `agentline doctor` (D11) names them.
