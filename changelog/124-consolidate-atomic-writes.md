- Consolidate atomic-write discipline behind one module. `src/lib/atomic-write.ts` exports three helpers — `writeIdempotent` (canonical tmp + fsync + rename, safe to call repeatedly), `writeJsonIdempotent` (JSON convenience over `writeIdempotent`, the previous `atomicWriteJson` shape), and `writeOnce` (exclusive create via `O_EXCL` + fsync, throws `EEXIST` on race). The old `src/config/atomic.ts` is deleted; `src/config/index.ts` re-exports the new names so callers that imported from the config-layer barrel still resolve. Eight production sites migrate to the new helpers (`src/state/{render-cache,stdin-cache,version-check-cache}.ts`, `src/doctor/fix.ts`, `src/tui/persist.ts`, `src/config/mutate.ts`, and the two new uses below).
- `src/lib/nerd-font.ts` `writeNerdFontStatus` switches from a hand-rolled `writeFileSync` + `renameSync` (no fsync) to `writeJsonIdempotent`. The signature becomes `async (stateDir, available) => Promise<NerdFontStatus>` — the function had no in-repo callers yet, so the sync→async change is a no-op for the current install path (the inline node script in `scripts/install.sh` still owns the live sentinel write).
- `src/state/backup.ts` `saveStatusLineBackup` switches from the hand-rolled `open(path, "wx")` + `fh.writeFile` + `fh.sync` to `writeOnce`. The "first writer wins" race semantics are preserved verbatim — `writeOnce` opens with `O_EXCL` and the existing `isEexist` catch is unchanged — and the redundant `mkdir` is dropped because `writeOnce` already does it.
- New `src/lib/atomic-write.test.ts` covers all three helpers: idempotent overwrite, buffer + string payloads, file-mode honouring, the rename-failure cleanup path (`fs.rename` mocked to throw EACCES, asserting no `.tmp` leak), `writeOnce` `EEXIST` on second call, and parent-dir creation on demand. Replaces the old `src/config/atomic.test.ts` (`atomicWriteJson` coverage subsumed by `writeJsonIdempotent`).
