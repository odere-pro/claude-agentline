- New `agentline preview` subcommand renders a sample statusline against a built-in stdin payload — no install, no piping, no host session. Supports `--theme`, `--config`, `--all-themes`, and `--minimal`/`--default` for previewing shipped templates. With no flags, prefers the user's saved config and falls back to the shipped default template so a fresh install renders a rich demo bar.
- `agentline init` gains `--preset {minimal,default,focus,power}` and `--scope {user,project}` flags. `--minimal` is preserved as a back-compat alias. Two new shipped presets ship under `templates/presets/` — `focus` (model + git + context + clock) and `power` (full default + thinking-effort, weekly-usage, block-timer). After a successful write the command prints next-step hints for `agentline preview` and `agentline doctor --fix`.
- Every subcommand now responds to `-h`/`--help` with command-specific usage text, dispatched through a new `HelpRequestedError` sentinel so each command's help body lives next to the parser. The top-level `agentline help` listing drops PR-number placeholders and leads with `agentline preview`.
- Friendlier error messages on `StdinParseError`, render error, and empty-stdin paths: stderr now includes one-line hints pointing at `agentline preview`, `agentline doctor`, and `agentline config` instead of bare diagnostics.
- README leads with `npx @agentline/cli preview`; `docs/install.md` retires `bash scripts/init.sh` from the per-project init recipe in favour of `agentline init --preset <name>`. The shell stub is kept as a compatibility shim so gate 04 stays green.
