- Two new rate-limit widgets — `block-reset-at` and `weekly-reset-at` — render the absolute wall-clock of the next reset (e.g. `resets 18:30`, `week resets 00:00`), complementing the existing countdown form (`block-reset-timer` / `weekly-reset-timer`). The new widgets reuse `blockEnd` and `weekStart` from `src/tokens/` for the reset moment and run the result through `formatClock` from `src/widgets/time/clock.ts`, so they inherit the same token grammar (`HH:mm`, `h:mma`, `HH:mm:ss`) and `options.tz` handling the `clock` widget supports — no new formatter. Default label is `"resets "` for block and `"week resets "` for weekly; `rawValue: true` suppresses the label. When the tokens snapshot is missing, the block widget falls back to "5h from now" (same arithmetic the timer uses); the weekly widget is purely clock-driven and always works. Catalogue entries declare three variants apiece (`time-24h`, `time-12h`, `seconds`) that mirror the new widget's actual format tokens (unlike the existing `clock` widget's strftime-style variants, which the tokenizer never wired up — out of scope for this PR).
