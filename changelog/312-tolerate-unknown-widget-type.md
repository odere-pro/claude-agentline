- fix: an unrecognised `widget.type` no longer discards the entire config. The schema's `widget.type` enum is generated from the widget catalogue, so a config naming a retired widget failed validation outright — and validation is all-or-nothing, so the statusline collapsed to a fallback single cell rather than losing just that one. PR #206 shipped exactly this break for the eight widget types it retired: an affected user saw their whole statusline vanish behind a cryptic `D03 config invalid` error. Two mechanisms written to handle the case were unreachable behind it — `agentline doctor`'s D11 check reported `config not loaded — see D03`, and the render pipeline's unknown-widget warning rows never fired. Validation now tolerates the unknown type (mirroring the existing "retired top-level keys are dropped, never hard-failed" rule), the render hides the cell silently, and D11 names the offending types. The warning rows are removed rather than revived: they would add physical rows and reintroduce the variable row count that corrupts the host's repaint accounting.
