- Split `src/tui/state.ts` (518 LOC, 30 functions) into three focused modules: `state.ts` (now 292 LOC — types, reducer, initial state, selectors, shared internals), `state-mutations.ts` (124 LOC — `moveCursor`, `moveWidget`, `deleteWidget`, `setOption`), and `state-picker.ts` (198 LOC — `openPicker`, `pickCategory`, `pickWidget`, `pickVariant`, `pickerBack`, `backToEdit`, commit transitions, option sanitisation). `reduce` in `state.ts` is now a thin dispatcher that imports the helpers; nothing observable changes for callers, but each unit fits well under the 800-LOC soft cap and the smell-report's <200-LOC-per-function guidance.
- `EditorState` is now a mode-indexed discriminated union: `EditorEditState` (mode: "edit") carries only the base fields; `EditorPickerState` (mode: "picker-group" | "picker-widget" | "picker-variant") additionally carries `pickerTarget` and `pickerDraft`. Reading those fields in edit-mode code is now a compile-time error — the prior shape allowed picker fields to surface as stale data when the editor was in edit mode. The `commit` / `backToEdit` transitions explicitly strip the picker-only fields when returning to edit, so a closed picker can't leak stale draft state.
- `src/tui/use-editor-input.ts` adds defensive `state.mode === "edit"` guards at the top of `handlePickerWidget` and `handlePickerVariant` for TS narrowing (the `useInput` dispatcher only routes there when in the matching picker mode; the guard is for the compiler, not the runtime). The reset-on-transition `useEffect` derives `pickerCategory` / `pickerWidgetType` nullable views so its dependency array stays accessible without re-narrowing each tick. `src/tui/app.ts#usedTypes` switches its dependency array to `[state]` since `state.pickerTarget` is no longer accessible unconditionally.
- `src/tui/state.test.ts` adds an `asPicker(state)` helper that narrows `EditorState` → `EditorPickerState` (throws if the test reaches an unexpected mode), and drops two assertions that asserted `pickerDraft === {}` against edit-mode states — that contract is now expressed by the type system, not by a runtime equality check.
