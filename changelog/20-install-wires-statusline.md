Make agentline a one-command install: `scripts/install.sh` now installs (or `npm link`s with `--from-source`) the package, seeds the user config from the bundled default, copies the four shipped themes into the user's themes dir, and writes a working `statusLine` entry into Claude Code's settings file. It's idempotent (re-runs are a byte-for-byte no-op), `--dry-run` describes every action without touching disk, and it refuses to overwrite a non-agentline `statusLine` unless `--force` is set. `scripts/uninstall.sh` mirrors the cleanup: only removes themes / config that still byte-match the shipped originals (preserving user edits), strips the `statusLine` entry only when it still references agentline, and supports `--purge` and `--dry-run`. Thirteen new integration tests cover the round-trip, idempotency, dry-run parity, foreign-statusLine refusal, and `--purge` behaviour.
