- Fix: `config.theme` is now actually applied at render time. Until this PR the field was stored but never loaded — `renderForFixture` (`src/render/fixture-runner.ts`) hard-coded `theme: null`, so selecting a theme had zero visible effect on the statusline and the `agentline config` live preview rendered uncoloured. A new `src/theme/resolve.ts` exposes `resolveConfiguredTheme(name, opts?)` (walks the same search path the theme browser uses — `${CLAUDE_CONFIG_DIR:-~/.config}/agentline/themes/` then the bundled `themes/` — loads the first match, never throws, returns `null` on miss / parse error / path-shaped name) and `themeDirectories(opts?)` (the search path), and `src/theme/command.ts` now reuses both instead of duplicating the bundled-dir lookup. `renderForFixture` resolves the theme when `options.theme` is `undefined`; explicit `null` opts out (the golden harness path); explicit `Theme` still pins. The TUI editor (`src/tui/main.ts`, `src/tui/preview.ts`) resolves `config.theme` once at startup in `runConfigCommand` and threads it to `Preview` so the live preview matches what the real bin renders. Existing goldens are unaffected — every recorded fixture has `config.theme: null` and `flags.noColor: true`. New tests: `src/theme/resolve.test.ts` (search-path ordering, user-dir precedence, traversal rejection, malformed-file fallthrough) and `src/render/fixture-runner.test.ts` (the dark theme's `#cc785c` accent reaches the ANSI bytes for the `model` widget; default palette wins when `theme: null`; unknown names fall back silently; explicit `options.theme = null` overrides resolution).
