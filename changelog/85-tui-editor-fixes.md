- The help overlay in `agentline edit` now closes only on `Esc`, `?`, or `q`. Previously any key — arrow, verb, typed character — both dismissed the modal and acted on the editor below, so users hitting an arrow to scroll the help would find their cursor had moved in the preview as well. New pure helper `shouldDismissHelp(input, key)` exported from `src/tui/main.ts` so the dismiss rule stays unit-testable without an Ink test harness; five test cases in `src/tui/main.test.ts` cover `Esc`, `?`, `q`, the ignored verbs (`a`, `s`, `Q`), and bare arrows. The overlay's footer line is updated to reflect the explicit close keys.
- Save-status message documents the pull-based reload model. `agentline edit` now reports `saved → <path> (reloads on next prompt)` instead of `saved → <path>`, so users know the on-disk change won't appear in the statusline until the next render tick (the host fires the bin per prompt).
- New `src/state/stdin-cache.ts`: every successful live `agentline render` writes its parsed `StdinPayload` to `${CLAUDE_CONFIG_DIR}/agentline/state/last-stdin.json` (or `~/.config/agentline/state/…`) using the existing atomic write-temp + fsync + rename pattern. Best-effort — write failures are swallowed so a read-only home or permission glitch never breaks the render hot path. Replays (`--fixture`) and pinned-config runs (`--config`) are deliberately excluded so the golden harness stays deterministic. Synchronous read (`readLastStdinSync`) — the file is small, the TUI bundle is only loaded on `agentline edit`, and Ink renders need a sync call. Six tests pin round-trip, missing-file, malformed JSON, version mismatch, swallowed write errors, and the supplied-clock branch.
- Editor preview switches from a hard-coded demo fixture to real cached values. `src/render/demo-fixture.ts` is retired and replaced by `src/render/preview-fixture.ts`, which lazily resolves a `PreviewMode` once per editor session: a `real` mode that rebuilds the same `WidgetContext` the live render saw (stdin payload + sync-loaded session / tokens / git snapshots) or a `label` mode that emits each widget's own `type` as its preview text. The decision is made by checking for the stdin cache file. New `renderWidgetLabel` export in `src/widgets/render-widget.ts` builds the label-only cell so the preview can call it directly without threading a flag through every widget's `render` function. The picker and the editor's live preview both call the same `previewWidget(type, options?, opts?)` API; consumers were updated to import from `preview-fixture.js` instead of `demo-fixture.js`. Test fixtures (`preview-model.test.ts`, `preview.test.ts`) pin a real-mode `PreviewMode` so existing branch / model assertions still pass; a new `buildPreview — label-only fallback` test covers the no-cache path, and `preview-fixture.test.ts` covers both modes end-to-end across every built-in widget.
- Verification: 1056 unit tests passing (across 107 files; +10 net from the new ESC-dismiss, stdin-cache, preview-fixture, and label-mode tests; -3 from retiring the demo-fixture tests), 10 gates green, 2 skipped (16 / 18 — pre-existing).
