- TUI save lifecycle hardened: a SIGTERM that arrives while `agentline edit` is mid-save no longer kills the atomic `temp Ôćĺ fsync Ôćĺ rename` between fsync and rename, which previously left an orphan `.tmp.<hex>` file in the agentline config directory. `enterAltScreen` accepts an optional `awaitBeforeExit` callback that the SIGTERM handler awaits before calling `process.exit`; the editor's `mountEditor` wires it to a shared `SaveTracker` so the handler reads the same in-flight promise the save callback owns.
- Replace the editor's `saveInFlight` boolean ref with a `SaveTracker { inFlight: Promise<void> | null }` object passed as an `App` prop. Re-entrant `onSave` calls now return the existing in-flight promise instead of dropping the keypress silently, and the handler clears `inFlight` back to `null` on settle so subsequent saves can claim the slot. Existing `useInput` save-key guard and `handleEdit` re-entry check read the same tracker, no behavioural change for the synchronous case.
