- The render path now wraps overflow instead of truncating mid-cell. `composeLines` (plain mode) packs visible cells greedily into output lines that fit `options.width`; when a cell would push the running total past the width, the composer starts a fresh output line. The first cell on a line always stays put — a single oversized cell overflows the terminal but is never silently dropped — and the total output line count is bounded by `MAX_LINES = 3` so a wildly-broken config can't fill the screen. Powerline mode is intentionally unchanged: chevron continuity across a wrap is a separate problem that v0.1.0 keeps out of scope. Seven new tests in `src/render/compose-overflow.test.ts` cover the obvious shapes (fits, two-line wrap, mid-string protection, oversized solo cell, hard cap at three lines, single-cell no-op, styling preserved across the wrap).
- Unknown widget types now surface as one warning line per offender below the rendered statusline. The pipeline collects every `config.lines[*].widgets[*].type` for which the registry has no entry, deduplicates and sorts the set, and appends `agentline: unknown widget type '<type>'` lines (themed `danger`) past the `MAX_LINES` cap. Warnings are bounded separately at `MAX_WARNING_LINES = 6` so a fat-fingered config that mistypes a dozen names doesn't bury the screen — the alphabetically first six surface and the rest are silently dropped. Renaming or removing a built-in no longer means the line silently shrinks; the user sees what's missing. The channel is a generic `Segment[][]` append point, so future warning sources (load-time validation failures, deprecated types) can land on the same surface without touching the rendering. Four new tests in `pipeline.test.ts` lock the dedupe / cap / no-false-positive shape.
- The Nerd-Font glyph separator is back to a plain `U+0020` space. The previous `U+2009` thin space rendered fractional-width in monospace terminals and visibly kerned against the width-2 PUA glyphs (the cell stretched ~half a column past where the user's eye expected it). A regular space lines up cleanly. The render-widget test that pinned ` ` has been refreshed; a new `src/widgets/glyph-width.test.ts` enforces every catalogue glyph is exactly one codepoint and lives inside the Nerd Font PUA ranges (BMP `E000-F8FF` or Supplementary `F0000-FFFFD`) so a stray emoji-sequence or zero-joiner never reaches the cell.
- Retired-widget sweep: the `flex-separator` example was still loitering in `agents/agentline-configure.md`'s config snippet despite the widget being deregistered in the catalogue trim. Removed.
