- fix: `tokens-cached`, `context-cached`, and `context-percentage`'s `showCached` postfix now all report the cached portion of the **current** context window, read from the host's `context_window.current_usage` block, instead of summing per-turn cache reads across the whole transcript. Every turn re-reads essentially the entire prompt cache, so the old cumulative sum grew without bound — a real 484-turn session rendered `163M` against a true cached context of `322k`. `tokens-cached` also read the live transcript file while `context-percentage` read the frozen stdin payload, so one statusline could report cache two incompatible ways. `tokens-cached` is consequently no longer an accumulator and declares no reset axis (`options.reset`, `inputGlyph` and `outputGlyph` no longer apply to it).
