- Fix `context-percentage` / `context-percentage-usable` / `context-bar` reporting 999%: they used to aggregate every prompt's input tokens across the whole session and divide by the model's context window, which exceeds the window many times over on a long Claude Code session. The stdin adapter now extracts `context_window.current_usage` (`input_tokens + cache_read_input_tokens + cache_creation_input_tokens`), `context_window_size`, and Claude Code's pre-computed `used_percentage`, and the widgets prefer that live snapshot — falling back to the cumulative aggregate only when stdin omits `context_window` (older Claude Code, golden fixtures). Also align the editor preview's default widget colour with the live render: the cell's own theme-resolved `fg` now wins over the picker family accent, so chips paint the same colour they will in the bottom statusline; user `widget.fg` overrides still take priority and the family accent stays as the label-mode fallback.
