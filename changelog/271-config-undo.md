- Add `agentline config undo` — user-reachable, single-level rollback of the last config change (brainstorm 04-self-healing §3, Bug 4):
  - Until now a `config.json.bak` only existed after `doctor --fix` overwrote a corrupt config; `config widget` mutations and TUI editor saves had **no** rollback. Both now back up the prior config to `config.json.bak` before the new config lands, through the one shared atomic-write seam (`backupAndWriteConfig`) — no second write mechanism, honouring the "atomic config writes" / one-X contract.
  - **`agentline config undo`** restores that backup atomically. Single-level (restores one step back, not a multi-level history); the backup is left in place so a re-run restores the same bytes (it is not a redo). Prints a one-line confirmation with the restored path on success; on "nothing to undo" it prints a clear message and exits non-zero rather than crashing. `config undo --help` prints help and does not act.
  - The backup unifies on the same `config.json.bak` path `doctor --fix` already writes (distinct from the host-statusLine `settings-backup.json`, which is untouched), so undo works after a repair too.
- Wired into `agentline config` alongside `widget` / `refresh` / `init`; documented in `docs/cli.md` and the `agentline-configure` skill.
