- Drop `/var` from the `CLAUDE_CONFIG_DIR` system-directory deny-list introduced in commit `eb0d433`: macOS resolves `$TMPDIR` to a `/private/var/folders/...` path (and so does `tmpdir()`), and `/var/tmp` is a valid temp root on most Unixes, so a blanket `/var` rejection blocked every install integration test that runs under a tmpdir; `/etc`, `/usr`, `/bin`, `/sbin`, and `/boot` remain rejected because they contain only system content.
