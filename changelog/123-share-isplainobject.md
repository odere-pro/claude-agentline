- Export `isPlainObject` from `src/lib/object.ts` and consume it from every site that previously hand-rolled the `typeof === "object" && !== null && !Array.isArray()` triplet before casting a `JSON.parse` result to `Record<string, unknown>` — `src/state/{render-cache,stdin-cache,backup,version-check-cache}.ts`, `src/tokens/transcript.ts`, `src/update-check/fetch.ts`, `src/render/fixture-runner.ts`, `src/stdin/index.ts`, `src/session/{index,auth-file}.ts`, and `src/doctor/checks.ts`. The narrowing the helper provides removes the trailing `as Record<string, unknown>` cast from each site; `src/config/merge.ts` drops its private copy of the same predicate.
- Fix a latent bug in `src/doctor/checks.ts`: the D02 check cast the parsed `~/.claude/settings.json` to `Record<string, unknown>` without first confirming it was an object, so a settings file containing a top-level JSON primitive (e.g. `"foo"`) would have read `"statusLine"` off a string instead of warning the user. The new guard converts that case into the existing "settings file is not a JSON object" warn outcome alongside the missing/unreadable path.
- New unit test `src/lib/object.test.ts` covers the accept set (plain literal, `Object.create(null)`), the reject set (`null`, arrays, primitives, class instances, `Map` / `Set` / `Date`), and the TS narrowing — plus tightening assertions for the existing `pickString` / `pickStringArray` / `pickEnum` helpers that previously had no dedicated test file.
