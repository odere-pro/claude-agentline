- Rework the `agentline config` TUI reducer (`src/tui/state.ts`) into the foundation for the redesigned editor: `EditorState` gains a `mode` (`edit` / `picker` / `options`) and a `pickerTarget` (`insert` / `replace`); the cursor is now genuinely two-dimensional and capped at `MAX_LINES` (3). New actions — `move-cursor` (slide the selection, within a row via `dx` or across rows via `dy`), `move-widget` (carry the selected widget, appending it on a destination row and creating a new row when moving down off the last one if there's headroom), `open-picker` / `close-picker` / `apply-picker` (insert at the cursor or replace the selection), `open-options` / `close-options` / `set-option` (merge one option onto the selected widget; prototype-polluting keys are refused), and `mark-dirty`. The legacy single-line actions (`navigate`, `select-widget`, `add`, `delete`, `set-type`, the widget toggles, `mark-clean`) still work, so the current Ink view is unaffected; the view rewrite that drives the new actions (live preview, picker overlay, options sheet, refreshed keymap) follows. 21 new reducer unit tests.
