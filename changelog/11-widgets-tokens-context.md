Land the eight token / cost / speed widgets and the four context widgets, plus the data layer they share. The new `src/tokens/` module embeds a versioned USD-per-Mtoken pricing table (whose version is exposed to `agentline doctor`), a per-model context-window lookup (200k for the standard families, 1M for the `[1m]` variant), and a JSONL transcript reader with a process-wide cache keyed by `(path, mtime, size)` and the §O3 eviction policy (5 h idle / 32 MB total). The render-tick resolver `loadTokensSnapshot()` reads the transcript once and freezes a `TokensSnapshot` widgets index into; widgets stay pure and never touch the filesystem during `render()`. `tokens-input`, `tokens-output`, `tokens-cached`, and `tokens-total` each declare a reset axis (`session` / `block` / `day` / `week` / `model` / `effort`) that filters events before summing — mixed-axis aggregation is forbidden per §8.4, and a typo on `options.reset` falls back to `session` so the renderer can never crash on a stray value. `cost` prices each model independently inside the window so a block that mixes Sonnet and Haiku is priced correctly. `input-speed`, `output-speed`, and `total-speed` report rolling tokens/second over a 60 s window (configurable via `options.windowSec`, capped at one hour). `context-length` reports the session's used+cached count, `context-percentage` and `context-percentage-usable` divide by the model's window (or 80% of it) and colour-grade against the `tokens-low / tokens-mid / tokens-high` palette roles at the 60 / 80% boundaries, and `context-bar` renders a fixed-width bar (default 12, configurable filled/empty glyphs) coloured by the same roles. 68 new tests; 302 total passing.
