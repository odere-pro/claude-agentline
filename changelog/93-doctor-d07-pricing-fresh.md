- Implement doctor check D07 — pricing freshness. The check parses `PRICING_TABLE_VERSION` from `src/tokens/pricing.ts`, reports `pass` while the embedded pricing table is within 90 days of `now`, `warn` once it crosses the threshold (with the version date and age in days surfaced in the message), and `warn` on a malformed or calendar-invalid version. Future-dated versions are treated as fresh and reported as `0d old`. The check remains report-only — no `--fix` action — per spec §9.2. Also fix gate-17-keymap-coverage on Windows: the gate now imports `dist/keys.mjs` via a relative path from its temp work directory instead of an absolute MSYS-style path, which the Node ESM resolver was misreading as a drive-root on Windows (`/d/a/…` → `D:\d\a\…`) and failing with `ERR_MODULE_NOT_FOUND`.
