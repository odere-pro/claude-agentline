- Backup file race fixed: `saveStatusLineBackup` in `src/state/backup.ts` now opens the target with `O_EXCL` (`fs.open(target, "wx")`) so two concurrent `agentline doctor --fix` (or install + doctor) runs can never both write the file. The earlier `pathExists`-then-`atomicWriteJson` pattern had a TOCTOU window that let the loser clobber the winner's snapshot of the original `statusLine`, defeating the "first install wins" invariant.
- `readLastStdinSync` in `src/state/stdin-cache.ts` now reconstructs the cached `StdinPayload` via `adaptStatuslinePayload` (the same adapter the live stdin path uses) instead of casting `payload as StdinPayload`. A stale cache from an older agentline version whose payload shape has drifted will fail validation cleanly rather than surface as a half-populated object.
- Add `pickEnum<T extends string>(obj, key, allowed)` to `src/lib/object.ts` for the case where a consumer wants exhaustive handling of a known-enum field; document the known value sets for `StdinPayload.outputStyle` / `vimMode` / `thinkingEffort` in their JSDoc while keeping the field types as `string` for forward-compat (the `thinking-effort` widget intentionally passes unknown levels through uncoloured rather than hiding them).
- Add `isEexist(err)` alongside `isEnoent(err)` in `src/lib/fs.ts` for callers that need to distinguish "already exists" from other open failures.
